Archive of Empirical Dynamic Programming

How does a work get into the Archive?

  • An archive item is a folder named using the convention Author1Author2…JournalYYYY in the AEDP github repository. For example, WolpinJPE1984 is the folder for Wolpin (1984).
  • A BibTex citation in a file with extension .ark is placed in the folder. For example, the original (not necessarily current) contents of the ark file in WolpinJPE1984 come from here. The extension is changed to .ark.

Is AEDP Just an Elaborate Bibliography?

    Yes and no.
  • Details about the DP model are added to the citation in the ark file. For example, WolpinJPE1984 can have this additional attribute: 
     clock = {NormalAging {("t",30)}}
    which says the model has a finite horizon clock over 30 periods.
  • Works are classified by Research Area, and each area is a "branch" in the git repository. Each area has curators to ensure a coherent organization. Areas can be split, merged, and added. For example, WolpinJPE1984 has the attribute area = {Development}
  • Code, data and other supplementary information can be added to the item's folder.

How can it help me understand empirical DP or do it myself?

  • Based on the information available in the .ark file, starter code for solving the paper's model(s) using niqlow is generated.
  • Users can push edits of the starter code to the repository. Eventually, code can qualify for includion in the niqlow replications folder.
  • Github Issue and Discussion posts can be used to ask for help by people in the AEDP community.

What Criteria are used for Including a Work?

  • Includes one or more dynamic programs as part of a model
  • Published:
    in an Economics-related journal or volume archived in JSTOR.org
  • Computed:
    results not based solely on closed form solutions
  • Non-trivial:
    $\delta\gt 0$ (static optimization only excluded)
  • Empirical:
    predictions of the DP are related to external data through calibration or estimation
