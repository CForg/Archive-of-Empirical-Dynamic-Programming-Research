How does a Work gets into the Archive?

  • An item begins as a folder named using the convention Author1Author2JournalYYYY in the AEDP github repository.
    For example, WolpinJPE1984 is the folder for Wolpin (1984).
  • A BibTex citation in a file with extension .ark is placed in the folder.

How is AEDP More Than an Elaborate Bibliography?

  • Details about the the DP model are added to the citation in the ark file.
  • These changes and additions can be made by:
    a AEDP Team member
    Any user of github through a pull request
    Anyone on the internet by filling out this form
  • Works are classified by Research Area, and each area as a "branch" in the git repository. Each areas has curators to ensure a coherent organization. Areas can be split, merged, and added.

How can it help me understand empirical DP or do it myself?

  • Code, data and other supplementary information can be added to the Work's folder.
  • Based on the information available in the .ark file, starter code for solving the paper's model(s) using niqlow is generated.
  • Other packages like VFI Tookkit can also add starter code folders.
  • Users can push edits of the starter code to the repository. Eventually, code can qualify for includion in the niqlow replications folder.
  • Github Issue and Discussion posts can be used to ask for help by people in the AEDP community.

What Criteria are used for Including a Work?

  • Includes one or more dynamic programs as part of a model
  • Published:
    in an Economics-related journal or volume archived in JSTOR.org
  • Computed:
    results not based solely on closed form solutions
  • Non-trivial:
    $\delta\gt 0$ (static optimization only excluded)
  • Empirical:
    predictions of the DP are related to external data through calibration or estimation
